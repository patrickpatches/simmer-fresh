Design improvements — regardless of colour direction

Fix now

Replace the emoji icon

⚠️ is an OS-rendered emoji — it looks different on every Android device and breaks the app's SVG-only icon language. Options 1 and 2 both replace it: a flame (culinary-specific, signals heat/care), or an herb/leaf (natural, calm). Both are custom SVGs at 13px, consistent with everything else in the prototype.

Fix now

Shorten the header label

"WHAT TO KNOW BEFORE YOU START" is 7 words in all-caps at 10.5px — the longest uppercase label in the app. It creates visual noise before you've read a single word. "BEFORE YOU START" or "A note before you begin" (serif italic) both communicate the same thing with half the weight.

Both directions

Full-width colour band > left border

A 3px left border is a whisper. A full-width colour band is a statement. The block contains genuinely important information — scrambled eggs ruin the dish. The design should match the stakes. Both options use a coloured header band so the visual weight matches the content weight.

Both directions

Bullet spacing + contrast

The current 8px gap between items reads tight at 12.5px body text. Both new variants use 10px. Also: the current bullet dots at 5px on a very light surface are barely visible — both new variants use higher-contrast bullets (rust or olive) that actually guide the eye between items.
